What is Git and Why I should use it.

Git is actually a memory card for your code

It is a "must have skill" for nowadays developers. Start deep diving into it whith this article by www.nobledesktop.com

The one book you should read about Git

visit https://git-scm.com and read online or download a copy to your computer.

Open command line

Change your directory to:

cd C:/Users/user/my_project

To create a new repository, type:

git init

A subdirectory named .git will be created.

If you want to start version-controlling existing files: 

git add *.c
git add LICENSE
git commit -m 'Initial project version'

If you want to clone an existing repository, type: 

git clone 
https://github.com/yourusername/repositoryname

View the commit history

git log

Ammend commits

git commit --amend

Unstage a staged file

git reset (commit id)

Git branching

Create a new branch

git branch (name)

Switch to new branch

git checkout -b (newBranch)

This is a shorthand for:

git branch (newBranch)
git checkout (newBranch)

Switch back to main branch

git checkout main

Merge branch into main

First checkout to your main branch, then:

git merge (newBranch)
